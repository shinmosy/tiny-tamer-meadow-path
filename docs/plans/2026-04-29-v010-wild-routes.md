# Tiny Tamer v0.1.0 Wild Routes Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Upgrade Tiny Tamer from prototype into a playable vertical slice with coherent art direction, route-based adventure map, quest loop, save/load, richer menus, Dex/team utility, and deeper battle foundation.

**Architecture:** Keep Godot 4 Web export structure. Add focused autoload managers for settings/save/quests/theme while preserving existing `Main.gd` router. Implement v0.1.0 in safe phases so the game remains exportable after every phase.

**Tech Stack:** Godot 4.3, GDScript, SVG/vector assets, generated WAV audio, Web export to `public/`, Vercel static headers.

---

## Scope Pillars

1. **Art Direction v1:** cozy field-journal RPG style, consistent palette, UI components, title/menu redesign.
2. **Wild Routes Map:** route-based map flow with Meadow Camp, Bloomtrail Path, Whisper Forest, Bluebell River, Cragpeak Foothill, Cave Gate.
3. **Progression Loop:** quest chain, save/load, rewards, NPC state dialogue.
4. **Creature Utility:** Dex detail, 3-slot team foundation, active companion, roles/types/skills.
5. **Battle v2 Foundation:** Fight/Team/Bag/Capture/Run command layout, type feedback, status-ready structure, smarter AI baseline.
